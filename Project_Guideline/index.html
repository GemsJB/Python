3

Learning Outcomes Addressed

On successful completion of this project, you will demonstrate the ability to:

  • Analyse a real-world problem and design a modular Python solution
  • Apply OOP principles — classes, encapsulation, inheritance, and polymorphism
  • Implement persistent data storage using file I/O and MySQL (mysql-connector-python)
  • Handle runtime errors gracefully using try/except and custom exceptions
  • Evaluate your solution against functional and non-functional requirements
  • Present and defend your system clearly to an academic audience
4

Technical Requirements

🏗 OOP Design
  • Minimum 3 well-defined classes
  • At least one inheritance relationship
  • Encapsulation via private attributes + getters/setters
  • At least one polymorphism/method override example
💾 Data Persistence
  • File I/O using the with statement (read & write)
  • MySQL CRUD via mysql-connector-python
  • Data validation before all writes/inserts
🛡 Error Handling
  • All risky operations in try/except/finally
  • At least one custom exception class
  • User-friendly error messages (no raw tracebacks)
🖥 User Interface
  • Console-based menu-driven interface
  • Clear prompts, feedback, and exit option
  • All user inputs validated before processing
  • GUI is optional bonus
✅ Code Quality

PEP 8 compliance  ·  Meaningful variable and function names  ·  Inline comments explaining logic  ·  Minimum 2 reusable functions in addition to class methods  ·  Appropriate use of lists, dicts, tuples, or sets

Academic Integrity Policy

All submitted work must be the student's own original work. The following constitute academic misconduct and will be handled under KCA University Academic Regulations:

  • Copying code from another student (plagiarism)
  • Submitting AI-generated code without acknowledgement or demonstrable understanding
  • Sharing source code with other students prior to submission
  • Reproducing code directly from online sources without citation and modification
